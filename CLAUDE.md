# プロジェクト概要

これはAIプログラミングワークショップ用の **Flask アプリ** です。
参加者はノンエンジニア（企画職）であり、プログラミング経験がありません。

## ファイル構成と改修レイヤー

| ファイル | レイヤー | 役割 |
|---------|---------|------|
| `app.py` | **Backend** | サーバー側の処理。URLごとの動作やデータベースの読み書きを定義 |
| `init_db.py` | **DB** | データベースのテーブル構造を定義。カラムの追加・変更はここ |
| `posts.db` | **DB** | SQLite データベースファイル（直接編集しない） |
| `templates/base.html` | **UI** | 全ページ共通のHTML枠（ヘッダー、タイトルなど） |
| `templates/index.html` | **Frontend** | 投稿一覧画面のHTML |
| `templates/create.html` | **Frontend** | 新規投稿フォームのHTML |
| `templates/edit.html` | **Frontend** | 編集フォームのHTML |
| `static/style.css` | **UI** | 色・フォント・余白などのデザイン設定 |

## 改修レベルの基準

要求に対して「どの程度の改修が必要か」を以下の基準で判断してください。

| 改修レベル | 変更範囲 | 具体例 |
|-----------|---------|--------|
| **UI（小）** | `style.css` のみ、または HTML の表示部分のみ | ボタンの色を変える、フォントサイズを変える、余白を調整する |
| **Frontend（中）** | `templates/` の HTML ファイル（複数の場合あり） | 入力チェックを追加、レイアウトを変更、表示形式を変更 |
| **Backend（大）** | `app.py` の修正が必要 | 新しいURL（ルート）の追加、データの加工ロジック変更 |
| **Backend + DB（大）** | `app.py` + `init_db.py` + `templates/` の複数ファイル | 新しい項目（カラム）の追加、新しいテーブルの作成 |

## 回答時のルール

- **日本語で回答**すること
- 改修の指示を受けたら、**改修レベル（UI小 / Frontend中 / Backend大 / Backend+DB大）** を最初に提示すること
- **変更対象ファイル**を一覧で示すこと
- 参加者はプログラミング初心者なので、**専門用語は避け、平易な日本語で説明**すること
- ファイルを変更する前に、何をどう変えるかを説明してから変更すること
