以下の要求について、このプロジェクトの改修見積もりを行ってください。

## 要求内容
$ARGUMENTS

## 出力フォーマット

以下のフォーマットで見積もりを回答してください。

### 📊 改修見積もり

| 項目 | 内容 |
|------|------|
| **改修レベル** | UI（小）/ Frontend（中）/ Backend（大）/ Backend + DB（大） のいずれか |
| **変更ファイル** | 変更が必要なファイルを一覧で列挙 |
| **変更ファイル数** | 合計数 |

### 📝 変更内容の詳細

変更が必要なファイルごとに、以下を記載：
- ファイル名
- そのファイルで何を変更するか（1〜2行で簡潔に）
- そのファイルの改修レイヤー（UI / Frontend / Backend / DB）

### 💡 判断の理由

なぜこの改修レベルと判断したかを、以下の基準に基づいて1〜3行で説明：
- `style.css` だけで済む → UI（小）
- `templates/` の変更で済む → Frontend（中）
- `app.py` の修正が必要 → Backend（大）
- `app.py` + `init_db.py`（DB定義）の変更が必要 → Backend + DB（大）
- 変更ファイル数が多いほど規模が大きい

### ⚠️ 注意点（あれば）

実装時に気をつけるべきこと、既存機能への影響、追加で確認が必要なことがあれば記載。
